ISP Router "Hacking"

I don't really like calling it hacking but whatever

So, as I said before, when working with my server, I wanted to get my SIP credentials so I could use my server for calls through the landline! Of course, to do that, I had two options, Either Go to my ISP and be like "hey can I get the credentials?" (They'd likely say no), or use a vulnerability when you factory reset the ISP router to get the data by force. Guess which one I used.

With that path chosen, I used a small oversight found on the router Vodafone gives customers most of the time (Not the modern one), and, of course, pretty much used it to get them. What was my process? Well...

  • 1. Unplug the Router from the Fiber Modem (2 separate devices)
  • 2. Factory Reset the router
  • 3. Using the factory default settings, access the admin panel
  • 4. Downgrade the router to a previous firmware version (They autoupdate anyway)
  • 5. Once downgraded, setup port sniffing on the admin settings to the traffic from the modem and router is sent to LAN port 1 too
  • 6. Set up WireShark on my laptop, and plug in the WAN cable again
  • 7. Profit! Creds obtained for everything

While I'm not sure if it's entirely legal, so far they haven't complained or even done anything soooo.

Also, while I wanted SIP creds for the landline, I also got stuff like PPPoE, The Router's Per user admin password, and more!

And yes, the router, once it connects to the ISP, reconfigures itself to have a Per user admin password, that only THEY know. kinda bs but oh well.
